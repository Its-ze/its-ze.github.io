Introduction

This project outlines the process I used to create and host this website using GitHub Pages. By leveraging GitHub Pages, I was able to create a simple, easy-to-update personal website without the need for expensive hosting solutions or complex server configurations.

Getting Started

To create this website, I used HTML and CSS to build each page and hosted the site using GitHub Pages, a free static site hosting service provided by GitHub. The setup process involved configuring a publishing source for the repository to make the site publicly accessible. The site updates automatically whenever I push new content or changes to the GitHub repository.

Why GitHub Pages?

GitHub Pages makes it easy to maintain and update my website using Git's version control features. With every update, the site is rebuilt and redeployed seamlessly, ensuring that any new projects, blog posts, or stories I add are instantly visible to visitors.

Steps I Followed

  1. Set up a GitHub repository to store all website files.
  2. Created the website layout using HTML and CSS.
  3. Configured GitHub Pages in the repository settings to publish the website.
  4. Used Git to commit and push changes, which automatically updated the live site.

Enhancements and Tools

To enhance the functionality and aesthetics of the site, I incorporated several tools and techniques:

  • Font Awesome: Added icons to make navigation more intuitive.
  • Google Fonts: Improved typography with custom fonts.
  • Responsive Design: Applied techniques to ensure the site looks great on all devices, from desktops to smartphones.
  • Dynamic Content Loading: Used JavaScript to load common elements like headers and footers dynamically, reducing redundancy and easing maintenance.
