Introduction

As I continue to refine my workflow, I've begun leveraging Git for both note-taking and website management. Using GitHub Desktop for version control and syncing, I can ensure my notes are securely backed up and that all changes to my website are intentional and properly tracked.

Setting Up Git for Note-Taking in Obsidian

Step 1: Create a Git Repository for Your Notes

  1. Create a new repository: Go to GitHub and create a new repository, such as gitnotes.
  2. Clone the repository: Open GitHub Desktop and clone the repository to a local folder on your machine.

Step 2: Integrate Git with Obsidian

  1. Install Obsidian Git plugin: In Obsidian, navigate to Settings > Community Plugins, search for Obsidian Git, and install it.
  2. Enable the plugin: After installation, enable the plugin to start using Git within Obsidian.

Step 3: Set Up Your Vault to Use Git

  1. Create a vault: In Obsidian, click on Open folder as vault and select the folder where you cloned your Git repository.
  2. Now, any notes you create or modify in this vault will be tracked by Git.

Step 4: Test Your Setup

  1. Create a test note: Add a new note in your Obsidian vault.
  2. Commit changes: Use the Obsidian Git plugin or GitHub Desktop to commit and push your changes to GitHub.
  3. Verify: Check your GitHub repository to ensure the note has been uploaded.

Managing My Website with GitHub Desktop and VS Code

Step 1: Clone the Website Repository

  1. Clone the repository: In GitHub Desktop, clone your website's repository to your local machine.

Step 2: Open the Repository in VS Code

  1. Open with VS Code: In GitHub Desktop, click Open in Visual Studio Code.

Step 3: Make and Commit Changes

  1. Edit files: Use VS Code to make changes to your website files.
  2. Stage changes: In VS Code's Source Control panel, stage the changes you want to commit.
  3. Commit: Write a meaningful commit message and commit your changes.

Step 4: Push Changes to GitHub

  1. Push to GitHub: In GitHub Desktop, click Push origin to upload your commits.
  2. Your website will update automatically if hosted via GitHub Pages.

Why I Chose This Workflow

Utilizing Git for both note-taking and website management streamlines my workflow, ensuring everything is version-controlled and easily accessible. This setup provides peace of mind, knowing that all changes are tracked and can be rolled back if necessary.
